## MVC介绍  
![MVC](http://www.artima.com/images/dci/Figure2.jpg)    

### 历史  
MVC最初是由[Trygve Reenskaug](http://heim.ifi.uio.no/~trygver/index.html)创造([文章来源](http://heim.ifi.uio.no/~trygver/themes/mvc/mvc-index.html)),他将MVC引入Smalltalk-76。Steve Burbeck等将MVC应用到[(来源)](http://st-www.cs.illinois.edu/users/smarch/st-docs/mvc.html)Smalltalk-80中。  

现在MVC进化成了多种形式：[HMVC](http://en.wikipedia.org/wiki/Hierarchical_model%E2%80%93view%E2%80%93controller)、[MVA](http://en.wikipedia.org/wiki/Model%E2%80%93view%E2%80%93adapter)、[MVP](http://en.wikipedia.org/wiki/Model%E2%80%93view%E2%80%93presenter)、[MVVM](http://en.wikipedia.org/wiki/Model_View_ViewModel)等。

### 例子  
打开我的电脑，进入文件夹，选择文件夹的查看方式，有：缩略图(thumbnail)、平铺(Tiles)、图标(Icons),列表(List),详细信息(Detail),小图标(SmallIcons)。  
![视图切换](http://media-cache-ec3.pinimg.com/736x/e7/f1/32/e7f132fc93d3d1cacdfca4f7117fedcf.jpg)     

这里有两种元素：   

>+数据   
>+显示方式  

数据是一致的，显示方式可变。MVC(模型、视图、控制)是将数据(后端)和显示(前端)分离的有效方法。  

### M,V,C  
+ MODEL。数据模型或业务模型。存放数据信息或业务操作。    
+ CONTROLER。控制器。决定数据如何显示，负责接收视图的请求。做为视图和数据的桥梁。  
+ VIEW。视图。数据的显示方式。  

举个例子：  
一个手机，有外壳，电池，屏幕，主板等组装而成。这些构成元素就是“MODEL”；  
对手机进行组装，设置组装的大小、颜色等属于“CONTROLLER”的职责；  
而视图就是显示的方式，如滑盖型、直板型、翻盖型等，它做为“CONTROLLER”的仆人，为它工作。  

控制器C通过视图V来控制/组织模型M的显示方式。    
MVC之间的通信方式如下图(注，图来源于斯坦福大学公开课iOS教程)：  
![iOS中的MVC](http://media-cache-lt0.pinterest.com/736x/51/de/11/51de11bc41445b4cb590738c78932937.jpg)   
上图中模型与视图的关系是分割开的，模型通知控制器；但在另外一个MVC模型，如STRUCTS中，数据模型的改变一般直接通知视图。到底选择什么样的沟通机制，视应用场景和MVC架构而定。  


#### MVC在各语言领域里都有应用。这里所写的仅是我理解的笔记，会有偏颇与错误，请君纠正。   
附一个HELLO,WORLD的例子，摘自[stackoverflow](http://stackoverflow.com/questions/8497833/hello-world-in-mvc-pattern):  
将以下代码  

>    var arr = ["a","b","c","d"];   // this is an array, same as store or model
>    alert(arr[0]);                // this is controller
>    //and browser alert is a view.

用MVC方式表示：    
  
>
>    var M = {}, V = {}, C = {};
>        
>    M.data = "hello world";
>    
>    V.render = function (M) { alert(M.data); }
>    
>    C.handleOnload = function () { V.render(M); }
>    
>    window.onload = C.handleOnLoad;
>
